#Samurai-Fight
A local 2-player browser fighting game. Two players share one keyboard, each controls a character, and combat continues until one player's health hits zero — at which point the round resets.
Implemented:

Two-player local input (separate key bindings per player)
Health and damage system with instant round-reset on death
Frame-based animation and sprite rendering on HTML5 canvas
Basic collision detection between attack hitboxes and player hurtboxes

Stack: JavaScript, HTML5 Canvas, CSS
Scope: Single-screen, no levels, no AI opponent, no networking. Built as a learning project to understand game loops, sprite animation, and hitbox-based combat in vanilla JS.
